First crate with `unsafe` — every block carries an audited SAFETY comment.

- `desmos-rt::linux::EpollReactor` Linux epoll backend using hand-declared `extern "C"` bindings for `epoll_create1`, `epoll_ctl`, `epoll_wait`, and `close` (no `libc` crate). Handles architecture-specific `epoll_event` layout (packed on x86_64, padded on every other arch), surfaces `EPOLLERR` / `EPOLLHUP` as readable, and maps `EINTR` to a zero-event return so idle loops keep ticking. Linux-only integration tests verify: UDP socket register → read-ready, 1000 register/deregister cycles leak no fds, and deregistered sources stop delivering events.

O(1) `schedule(after_ms)` returns a `TimerId`, `poll(now_ms)` fires cascaded slots into a caller-owned `Vec<FiredTimer>`. Benchmark: 1 000 000 insert + pop in 8.9 ms (Apple M1 release, spec called for < 200 ms on x86_64).

Loopback round-trip and non-blocking behaviour unit-tested on every Unix; `bind_on_lo_reports_device` and empty-interface rejection Linux-only. `socket2` pulled with `features = ["all"]` so the Linux-only `bind_device` is reachable.
- `desmos-core::pipeline` single-interface plaintext packet pipeline closing out Phase 1. `forward_tun_to_udp` encodes a DWP Data header over an IP packet and punts it to the peer; `forward_udp_to_tun` decodes the header and writes the payload back to the TUN. `run_plaintext_linux(PlaintextConfig)` stitches a `LinuxTun` + `UdpSocket` together behind an `EpollReactor`, drains both sides on readiness, and treats malformed frames as dropped packets. Cross-platform pipeline stages and a loopback integration test with an in-memory `MockTun` exercise the full round trip without needing `CAP_NET_ADMIN`. `desmos-core` now depends on `desmos-proto` and `desmos-rt`.
- `desmos-cli::commands::up` real `UpCommand` replacing the stub. Parses `--mode`, `--tun`, `--listen`, `--peer` in both inline (`--flag=value`) and separate (`--flag value`) forms, routes `--mode plaintext` to `desmos-core::pipeline::run_plaintext_linux` on Linux, and surfaces a helpful error on every other platform. `desmos-cli` now depends on `desmos-core` and `desmos-proto`. 6 new unit tests cover flag parsing branches.
- `desmos-proto::crypto` wrapper layer over `ring` and `blake3` (first use of both). `crypto::aead::AeadKey` wraps ChaCha20-Poly1305 with `seal_in_place` / `open_in_place` and a `CryptoError` taxonomy that surfaces wrong-key, tamper, nonce, and short-buffer failures as `AeadFailed`; manual `Debug` impl redacts the key material. `crypto::hkdf` implements RFC 5869 HKDF-SHA256 directly on `ring::hmac` (extract + expand + derive) and passes the RFC 5869 Test Case 1 vector. `crypto::hash::Blake3` wraps `blake3::Hasher` with `new`/`update`/`finalize`/`hash`. `blake3` pinned to `=1.5.4` because 1.8+ pulls `constant_time_eq 0.4.x` which requires Rust edition 2024. `deny.toml` allow-list extended with `BSD-2-Clause`, `Unicode-DFS-2016`, `Unicode-3.0` to accept ring/blake3 transitive crates.
- `desmos-proto::crypto::x25519_field` hand-rolled X25519 scalar multiplication (RFC 7748 §5) ported from the public-domain TweetNaCl reference. 16-limb radix-16 signed field representation, constant-time Montgomery ladder, Fermat-based inversion. Needed because `ring::agreement::EphemeralPrivateKey::diffie_hellman` is move-consuming and `SecureRandom` is sealed, so ring cannot expose the reusable static X25519 key Noise IK requires. Verified against RFC 7748 §5.2 test vectors plus the iterated-once vector, and cross-checked against `ring::agreement` across 50 random rounds (strongest correctness guarantee).
- `desmos-proto::crypto::x25519::X25519PrivateKey` persistent, non-consuming X25519 private key wrapping a 32-byte seed. `generate` / `from_bytes` / `public_key` / `diffie_hellman(&self, peer)` / `to_bytes`. The move-consuming `EphemeralPrivateKey` from Task 15 has been replaced — every DH operation is now reusable, which is what Noise IK's four DH tokens (`es`, `ss`, `ee`, `se`) require.
- `desmos-proto::handshake` Noise IK state machine implementing the `Noise_IK_25519_ChaChaPoly_SHA256` variant. `SymmetricState` (chaining key `ck`, transcript hash `h`, current `AeadKey`, per-key nonce counter) with `new` / `mix_key` / `mix_hash` / `encrypt_and_hash` / `decrypt_and_hash` / `split` / `handshake_hash`; SHA-256 via `ring::digest` so MixHash and HKDF share a hash family; Noise big-endian nonce layout (4 zero bytes + 8-byte BE counter). `Initiator` and `Responder` structs run the two-message IK pattern (`-> e, es, s, ss` then `<- e, ee, se`), reach matching transport keys in one round trip, and expose `TransportKeys { send, recv, handshake_hash }` with a redacting `Debug` impl. `Responder` enforces an optional initiator whitelist, surfacing `HandshakeError::UnknownStatic` on mismatch. `HandshakeError` taxonomy covers `Crypto`, `UnknownStatic`, `BadMessage`, `WrongState`. 7 unit tests in `handshake::tests` cover SymmetricState (nonce layout, padded-name init, long-name hashing, encrypt/decrypt roundtrip with matching states, nonce reset on MixKey, deterministic split, transcript-divergence rejection) and 9 integration tests in `tests/noise_ik.rs` verify matching transport keys, initiator-static exposure, whitelist acceptance and rejection, tampered message 1 / 2 MAC failures, malformed-length rejection, the fixed-static self-consistency reference check, and five sequential handshakes reusing the same static keys — the whole point of the Task 15 X25519 refactor.
- `desmos-core::session` typestate wrapping the Noise IK handshake. `Session<Handshaking>` / `Session<Established>` / `Session<Rekeying>` / `Session<Closed>` markers; `encrypt_data` / `decrypt_data` exist only on `Session<Established>`, verified by a `compile_fail` doctest. `Session<Handshaking>::new_initiator` / `new_responder` wrap `desmos-proto::handshake::{Initiator, Responder}` and `advance(self, peer_msg, now_ms)` returns a `HandshakeOutcome::{NeedsMore, Established}` so the same API drives both sides. `Session<Established>` holds two `AeadKey`s + `AtomicU64` send counter + established-at timestamp + transcript hash, seals with a Noise-compatible nonce (4 zero bytes then BE seq) and `[session_id || seq]` AAD so packets cannot be replayed across sessions. `Session<Rekeying>` keeps the old keys for in-flight decryption while a fresh embedded handshake runs, then swaps to the new `Established` via `advance_rekey`. `SessionError` taxonomy: `Handshake` / `Crypto` / `CounterOverflow` / `WrongStep`. 13 unit tests cover the end-to-end handshake, bidirectional encrypt/decrypt, counter increment, cross-session AAD rejection, rekey trigger at exactly 120 s, rekey-state old-key decryption, close transition, and AAD / nonce layout checks.
- `desmos-core::session::manager::SessionTable` (`RwLock<HashMap<SessionId, Arc<Mutex<AnySession>>>>`) with `new` / `insert` / `get` / `remove` / `contains` / `ids` / `len`. `AnySession` is the erased-state enum that restores the typestate guarantee at the call site via pattern matching. 8 unit tests cover the basic insert/get/remove/contains/ids flows and a 10 000-iteration deterministic xorshift64 fuzzer that drives random insert/lookup/remove/contains ops against the real table and a reference `HashMap<u16, bool>`, asserting both stay in sync at every step (proptest remains blocked on the MSRV 1.75 wall so the fuzzer is our substitute).
- `desmos-core::session::rekey` with `REKEY_INTERVAL_MS = 120_000` and an `is_due(established_at_ms, now_ms)` helper; `desmos-core::session::keepalive` with `KEEPALIVE_INTERVAL_MS = 15_000` and the same helper shape. 3 unit tests each covering the pre-interval, post-interval, and clock-regression cases.
- `desmos-proto::antireplay::AntiReplayWindow` 128-bit sliding anti-replay bitmap, shape modelled after RFC 6479 §3. Tracks the highest accepted sequence in `u64` plus a `u128` bitmap where bit `k` represents `highest - k`; `check_and_update(seq)` accepts in-order and out-of-order within the 128-wide window, rejects duplicates and out-of-window via a typed `ReplayError { Duplicate, OutOfWindow }`. `would_accept(seq)` is the non-mutating probe for speculative pipeline stages; `highest()` and the `new` constructor round out the API. The first packet on a fresh window is always accepted (even `seq == 0`) thanks to an `initialised` flag that separates "never seen anything" from "zero is the current highest". 12 unit tests cover every branch (fresh init, in-order bursts, out-of-order within window, exact window edge at offset 127, one-past-edge rejection, duplicate at highest and within window, WINDOW_SIZE forward jump overflow branch, large forward jump clearing the window, `would_accept` agreement, and clone independence). 4 integration tests in `crates/desmos-proto/tests/antireplay.rs` verify the real window against a HashSet-based `Reference` model: in-order, reverse-order, a pathological manually-crafted stream, and a deterministic 10 000-stream xorshift64 fuzzer where each stream runs 64 packets biased across the five branches (near cursor, recent duplicate, big forward jump, ancient out-of-window, exact cursor duplicate) and every real decision must match the reference exactly — stronger than the Task 18 "no false accepts" acceptance bar.
- `desmos-core::pipeline::{forward_tun_to_udp_encrypted, forward_udp_to_tun_encrypted}` Phase 2 encrypted packet pipeline stages sitting alongside the Phase 1 plaintext path. Outbound reads an IP packet from the TUN, calls `Session<Established>::encrypt_packet` (new API returning `(seq, ciphertext)`), stamps the assigned sequence into the DWP header's `Seq` field (narrowed from u64 to u32; sessions rekey well before the upper half is relevant), and sends `[16-byte DWP header] [ciphertext + 16-byte Poly1305 tag]` to the peer. Inbound recvs the datagram, decodes the header, widens the 32-bit wire sequence back to u64, calls `Session<Established>::decrypt_data` which enforces the per-session anti-replay window before the AEAD, and delivers the plaintext to the TUN. Drops are silent on the wire but each error class increments a specific counter: `bad_header` for malformed DWP headers, `replay_drops` for anti-replay rejections, `decrypt_failures` for tag mismatches / wrong-session / tampered ciphertext.
- `desmos-core::pipeline::PipelineMetrics` atomic counter block (`packets_sent`, `packets_received`, `bytes_sent`, `bytes_received`, `bad_header`, `decrypt_failures`, `replay_drops`) plus `record_*` helpers and a `MetricsSnapshot` struct for lock-free reads. 4 unit tests cover zero-init, sent/received recording, and independent error counters.
- `desmos-core::session::Session<Established>` gains `encrypt_packet` (returns `(seq, Vec<u8>)` for the pipeline's header-stamping needs), an embedded `Mutex<AntiReplayWindow>` that guards `decrypt_data` so duplicates and out-of-window sequences are rejected before the AEAD runs, and a `Replay(ReplayError)` variant on `SessionError`. `Session<Rekeying>::decrypt_with_old` also runs the old session's window for symmetry. 3 new unit tests cover `encrypt_packet` returning monotonically increasing sequences, decrypt rejecting a replay of the same packet, and decrypt rejecting an out-of-window sequence after the window has slid past it.
- `crates/desmos-core/tests/encrypted_loopback.rs` end-to-end encrypted pipeline test with 5 scenarios: in-memory Noise IK handshake completes under 15 ms release / 600 ms debug (taking the fastest of three runs after a warm-up to filter cold-cache noise, since the hand-rolled X25519 is ~20x slower under the default debug profile), a full encrypt → UDP → decrypt round trip delivers an unchanged plaintext, replaying the exact same UDP bytes is dropped and bumps `replay_drops`, a tampered AEAD tag is dropped and bumps `decrypt_failures`, and 500 back-to-back 1 KB packets survive a real UDP loopback round trip inside 2 seconds (sanity check for the 500 Mbps acceptance claim; the real iperf3 verification runs at release time).
- `desmos-core::net` host interface discovery and Linux link-state watching. `net::iface::list()` returns `Vec<NetworkInterface>` with name, 6-byte MAC, `Vec<Ipv4Addr>`, `Vec<Ipv6Addr>`, an `IfaceFlags` bitfield decoded from `/sys/class/net/<name>/flags`, and the `operstate` string. Linux path combines `/sys/class/net` enumeration (MAC, operstate, raw flags via plain file reads) with a hand-declared `getifaddrs` / `freeifaddrs` FFI for per-interface IP addresses; non-Linux Unix falls back to `getifaddrs`-only and zeros the MAC until an AF_LINK backend lands, and Windows returns `IfaceError::NotImplemented` until Phase 6. `read_sa_family` compensates for the BSD `sockaddr` layout where `sa_family` is a u8 at offset 1 instead of a u16 at offset 0 — verified by the `list_contains_loopback_on_unix` test on macOS. 8 unit tests cover MAC parsing, flag decoding, `mac_string` formatting, `is_bondable` semantics, and cross-platform loopback discovery.
- `desmos-core::net::watcher` Linux-only netlink link-state subscriber. `NetlinkWatcher::new` opens an `AF_NETLINK` / `NETLINK_ROUTE` socket with `SOCK_NONBLOCK | SOCK_CLOEXEC`, binds it to the `RTMGRP_LINK` multicast group, and exposes `drain_once` for event pulls plus `AsRawFd` so the socket plugs straight into the Task 10 `EpollReactor`. `InterfaceEvent::{LinkUp, LinkDown}` carry interface name and kernel index; the parser walks `nlmsghdr` + `ifinfomsg` + `rtattr` TLVs to extract `IFLA_IFNAME` and the `IFF_UP` bit. Hand-declared FFI for `socket`, `bind`, `recv`, `close` — no libc crate. `Drop` closes the fd so the kernel removes the multicast subscription automatically. 5 unit tests cover `align4`, empty-buffer parsing, undersized-header rejection, socket open, and non-blocking drain on an idle system.
- `desmos-cli::commands::interfaces` real `InterfacesCommand` replacing the Task 5 stub. `desmos interfaces` walks `desmos-core::net::list` and prints an aligned `NAME / STATE / MAC / IPV4 / FLAGS` table with continuation rows for extra IPv4 / IPv6 addresses; `desmos interfaces --json` (and the global `--json`) emits a hand-rolled JSON array with per-interface `name / mac / operstate / ipv4 / ipv6 / flags`. Unknown sub-verbs (`enable`, `disable`, `reweight`) return exit 64 with a "not implemented yet" message pending the bonding engine in Task 25+. 2 unit tests cover JSON string escaping and the flag-label joiner.
- `desmos-core::bonding` round-robin bonding engine scaffold. `bonding::link::Link` wraps a `LinkId`, interface name, peer `SocketAddr`, static weight, and a coarse `healthy` flag (to be upgraded by Task 25's link-health state machine). `bonding::link::LinkTable` is an `Arc<Link>`-backed read-only snapshot with `all`, `healthy`, `len`, `is_empty`, and `get(LinkId)`; the engine swaps whole tables so pipeline threads never see a torn view. `bonding::strategy::BondingStrategy` trait (`Send + Sync`, `name() -> &'static str`, `schedule(packet, links) -> LinkSelection`) plus the `LinkSelection { One, Many, None }` enum that lets Task 23's `Replicate` strategy return multiple links from a single call. `bonding::strategy::RoundRobin` holds an `AtomicUsize` cursor and picks `healthy[cursor % n]` with `Relaxed` ordering — the cursor wraps cleanly at `usize::MAX` thanks to the modulo. `bonding::Engine` holds `RwLock<Arc<dyn BondingStrategy>>` + `RwLock<Arc<LinkTable>>` so `schedule` clones the `Arc`s under a short read lock and runs the strategy lock-free; `swap_strategy` and `swap_links` atomically replace the pointed-to values without disturbing in-flight schedules. 19 unit tests cover RoundRobin rotation, unhealthy-link skipping, zero-healthy and empty-table fallback to `LinkSelection::None`, cursor monotonicity even when dead links are skipped, 8-thread concurrent scheduling converging on the expected cursor value, engine strategy hot-swap observed on the next schedule, link-table hot-swap, and a 4-reader + 1-writer stress test flipping strategies 1000 times while readers keep scheduling without torn state.
- `desmos-core::bonding::reorder::ReorderBuffer<T>` out-of-order reorder buffer with gap timeout. Generic over the payload type so tests drive it with cheap primitives and the pipeline uses `Vec<u8>`. Tracks `next_expected`, a `BTreeMap<u64, Pending<T>>` for out-of-order arrivals, `window` (max seq distance before a force-skip), `gap_timeout_ms` (wall-clock wait before a head gap is declared lost), plus `delivered` / `lost` / `duplicate` counters. `push(seq, payload, now_ms)` returns the zero or more payloads ready for in-order release; `poll_timeout(now_ms)` checks the oldest pending entry's age and, if it has waited past the timeout, skips past the missing seqs and drains everything contiguous. An `initialised` flag starts the buffer from the first observed seq so a stream whose counter was reset by rekey is handled cleanly. 12 unit tests cover in-order delivery, out-of-order reorder, multi-gap drain, duplicate-at-delivered / duplicate-at-buffered rejection, window-pressure force-skip, gap-timeout firing, chain drain after timeout, pending-count accuracy, late-packet-after-force-skip handling, and monotonic `next_expected`. 4 integration tests in `crates/desmos-core/tests/reorder.rs`: 100 k in-order packets zero-buffered, a 9 600-packet shuffled-window stream with strict-order delivery verification, a deterministic workload that drops every 17th packet and asserts the lost counter matches exactly, and a per-push latency bench that sorts 100 k samples and asserts the p99 is under 5 ms in debug / 1 ms in release — the Task 22 acceptance bar ("< 1 ms") with criterion stubbed out because it is still blocked by the MSRV wall.
- `desmos-core::bonding::score::LinkStats` rolling per-link quality with Jacobson SRTT EWMA (α = 0.125, RFC 6298), rolling-window loss rate, and jitter as the stdev of the RTT sample window. `record_rtt(rtt_us)` seeds and updates the EWMA plus pushes into the sample + outcome deques; `record_loss()` pushes only into the outcome deque. `loss_rate()` returns 0.0 on a fresh window (a brand-new link is considered healthy until proven otherwise), `jitter_us()` returns 0 with fewer than two samples, and `score()` folds everything into a `LinkScore` composite `(1 - loss) · 10 000 / (1 + rtt_ms + jitter_ms · 2)` — higher is better, a 1 ms / zero-loss link scores ~5 000, a 100 ms / 10 % link scores ~88. `DEFAULT_WINDOW = 100` matches the TASKS.md acceptance "rolling loss over last 100 probes". 12 unit tests cover fresh-state reporting, first-sample seeding, EWMA convergence across a step change, loss-rate rolling-window eviction, zero jitter for constant RTT, stdev against a hand-computed vector, single-sample jitter = 0, good vs bad composite ordering, 100 %-loss clamping to zero composite, sample-window size cap, and `new(0)` panic.
- `desmos-core::bonding::probe::ProbeScheduler` per-link probe cadence + in-flight tracker. `interval_us` / `timeout_us` (defaults 500 ms and 2 s), monotonic seq allocator, `is_due(now_us)` returns `true` before the first issue then only after an interval has elapsed, `issue(now_us)` stamps the send time into an in-flight `BTreeMap<u64, u64>`, `on_ack(seq, now_us)` returns a `ProbeAck { seq, rtt_us }` if the probe is still in-flight or `None` if unknown, and `drain_timeouts(now_us)` sweeps the in-flight map for entries older than `timeout_us`, returns their seqs, and removes them so a late ack cannot double-count. 12 unit tests cover first-call-due, interval gating, monotonic seqs, in-flight accounting, ack-then-drain idempotency, unknown-seq rejection, already-acked rejection, and `new(0, _)` / `new(_, 0)` panics.
- `crates/desmos-core/tests/probe_and_score.rs` integration suite that drives a `ProbeScheduler` + `LinkStats` pair through a fake-clock probe cycle: 20 successful probes with identical RTT converge the EWMA to the exact sample value, a mixed 8-success / 2-timeout workload reports the expected 20 % loss rate, a varying-RTT stream yields non-zero jitter, a shrinking 5-slot rolling window transitions through 100 % → 40 % → 0 % loss as fresh successes push old losses out, and the scheduler cadence is verified against the configured 1 s interval at microsecond precision.
- `crates/desmos-core/tests/rr_bonding.rs` end-to-end bonded-tunnel integration test closing out Phase 2 bonding v1. Runs 3 in-process tests plus 1 `#[ignore]`'d Linux-only veth gate: (1) `round_robin_distributes_packets_evenly_across_two_links` schedules 1 000 packets and asserts a perfect 500/500 split across two links; (2) `bonded_stream_survives_encrypt_two_link_split_and_reorder` runs a real Noise IK handshake, encrypts 200 unique-payload packets via `Session::encrypt_packet`, uses `Engine::schedule` to route each one through one of two UDP loopback sockets, alternates non-blocking recvs on both sockets so the per-session anti-replay window sees seqs in monotonic order (greedy drain would push the 128-wide window past the laggard link's oldest seq and drop packets — caught the first run), and asserts zero loss plus every original payload arrives; (3) `single_link_control_case_has_no_loss` is the 1× baseline the ignored veth test compares against. The `rr_bonding_veth_throughput_linux_only` test delegates to `scripts/rr_bonding_veth.sh` via `DESMOS_E2E_VETH` env override.
- `scripts/rr_bonding_veth.sh` Linux-only helper that provisions two veth pairs inside fresh `desmos_rr_c` / `desmos_rr_s` netns, pins 20 ms one-way delay + 100 Mbit/s rate cap on each via `tc tbf` + `tc netem`, runs a single-link iperf3 baseline followed by a parallel two-link run, parses the receiver-side Mbit/s out of each iperf3 report, and asserts the bonded aggregate is ≥ 1.5× the baseline — the exact TASKS.md Task 24 acceptance bar. Idempotent: a trap-driven cleanup removes any leftover netns / interfaces on exit even when interrupted. Requires bash, iproute2, iperf3, and root (CAP_NET_ADMIN).
- `desmos-core::bonding::strategy::Weighted` stochastic weighted sampler using a shared `next_xorshift64` helper on an atomic u64 state cell. `Weighted::new` seeds with the workspace golden ratio constant; `Weighted::seeded(seed)` takes an explicit start for deterministic tests across multiple strategies. `schedule` sums the healthy link weights, draws `rand % total` and walks the cumulative-weight array. Zero-weight fallbacks map to a uniform pick so a config typo that zeroes every link still forwards traffic. 5 unit tests: name check, empirical distribution across 10 000 samples within 5 % of the configured 5/3/2 split, unhealthy-link skipping, zero-weight uniform fallback, single-link always-picked, and empty-table `None`.
